1. Project Overview

This is a Django web application structured with reusable apps: accounts and inventors. It supports authentication, user management, and inventory functionality.

2. Directory Structure

.
├── accounts/
├── config/
├── inventors/
├── jenkins_service/
├── Jenkinsfile
├── Makefile
├── manage.py
├── requirements.txt
  • accounts/: Authentication and user management logic
  • config/: Core Django configuration (settings, routing, WSGI/ASGI)
  • inventors/: Domain-specific logic and models
  • jenkins_service/: Jenkins-Docker integration setup
  • Jenkinsfile: Jenkins CI pipeline definition
  • Makefile: CLI automation for common tasks (e.g., test, build)
  • manage.py: Django’s CLI entry point
  • requirements.txt: Python dependencies

3. App Modules

accounts/

  • models.py: Custom user model or profile extension
  • authentication.py: Token/session management
  • serializers.py: DRF serializers for user data
  • views.py: Views for login, signup, etc.
  • urls.py: Routing for account endpoints
  • utils.py: Reusable helper functions

inventors/

  • models.py: Inventory models or objects related to invention
  • serializers.py: DRF serializers for inventory
  • views.py: API views or class-based views
  • urls.py: Routing for inventory-related endpoints

config/

  • settings.py: Project-wide settings including apps, DB, REST, etc.
  • urls.py: Root-level routing combining all app URLs
  • wsgi.py: WSGI entrypoint for deployment
  • asgi.py: ASGI entrypoint for async services

4. CI/CD with Jenkins

jenkins_service/
  • dockerfile: Jenkins + Docker setup
  • entrypoint.sh: Ensures correct Docker socket permissions inside Jenkins container
  • script.sh: Script to build and run Jenkins container
  • docs.html: HTML docs for Jenkins setup

Jenkinsfile

Defines your CI stages such as:

  • Installing dependencies
  • Running unit tests
  • Linting
  • Docker image build (optional)

5. Running the Project

Install Dependencies

pip install -r requirements.txt

Run Development Server

python manage.py runserver

Apply Migrations

python manage.py makemigrations
python manage.py migrate

Run Tests

python manage.py test

6. Docker Support

Docker is used to run Jenkins with access to the host Docker daemon for container builds.

Build Jenkins Docker Image

cd jenkins_service
docker build -t jenkins-docker .

Run Jenkins Container

docker run -d \
  --name jenkins-server \
  -p 8080:8080 -p 50000:50000 \
  -v jenkins_home:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkins-docker
