Jenkins with Docker Inside

This documentation explains how to build a custom Jenkins container with Docker pre-installed and usable from Jenkins pipelines using the host Docker daemon.

1. Dockerfile

Dockerfile

Base image: jenkins/jenkins:lts. Adds Docker and gosu, sets the proper group for Docker socket access.

FROM jenkins/jenkins:lts

USER root

RUN apt-get update && \
    apt-get install -y docker.io && \
    usermod -aG docker jenkins && \
    apt-get install -y gosu 

COPY entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

ENTRYPOINT ["/entrypoint.sh"]

2. Entrypoint Script

entrypoint.sh

Dynamically adds Jenkins user to the same group as the Docker socket to allow containerized Jenkins to access host Docker daemon.

#!/bin/bash
set -e

DOCKER_SOCK="/var/run/docker.sock"
if [ -S ${DOCKER_SOCK} ]; then
    DOCKER_GID=$(stat -c %g ${DOCKER_SOCK})
    
    if getent group ${DOCKER_GID} >/dev/null; then
        existing_group=$(getent group ${DOCKER_GID} | cut -d: -f1)
        if [ "${existing_group}" != "docker" ]; then
            usermod -aG ${existing_group} jenkins
        fi
    else
        groupadd -g ${DOCKER_GID} docker_host
        usermod -aG docker_host jenkins
    fi
fi

exec gosu jenkins /usr/local/bin/jenkins.sh "$@"

3. Build and Run Commands

script.sh

Build and run Jenkins container with Docker access.

docker build -t jenkins-docker .

docker run -d \
  --name jenkins-server \
  -p 8080:8080 \
  -p 50000:50000 \
  -v jenkins_home:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkins-docker
  • -v /var/run/docker.sock: Gives access to host Docker daemon.
  • gosu: Runs Jenkins as a non-root user inside container.

4. Notes

  • Ensure Docker is installed on your host machine.
  • This setup enables Jenkins to build Docker images and run containers.
  • Using gosu avoids root execution for security.
